# edu_rag 问题排查记录

> 时间：2026-07-07 | 环境：Windows 10，Python 3.10，EduRAG

---

## 问题 1：MySQL 数据库 `edu_rag` 不存在

**现象**：启动日志 `MySQL 初始化失败（功能受限）: Unknown database 'edu_rag'`，创建知识库失败。

**原因**：MySQL 服务正常运行，但未创建 `edu_rag` 数据库。`init_db()` 不会自动创建数据库，只能创建表。

**修复**：
```sql
CREATE DATABASE IF NOT EXISTS edu_rag CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
重启服务后 `init_db()` 自动创建 `knowledge_bases`、`documents`、`exam_records` 三张表。

---

## 问题 2：Ollama 服务未启动

**现象**：上传文档、QA 问答报 500 错误，日志显示 `Connection error` 或 `SSLError`。

**原因**：`.env` 中 `EMBEDDING_PROVIDER=api`，Embedding 调用需要 Ollama 服务运行在 `localhost:11434`，但 Ollama 未启动。

**修复**：
```bash
ollama serve
```
确认 bge-m3 模型已下载（1.2GB），API 正常返回 1024 维向量。

---

## 问题 3：`.env` 误设为 `EMBEDDING_PROVIDER=local` 导致 SSL 错误

**现象**：`get_embeddings()` 触发 `ssl.SSLError: [ASN1: NOT_ENOUGH_DATA]`。

**原因**：`EMBEDDING_PROVIDER=local` 时会导入 `sentence_transformers` → `datasets` → `aiohttp`，`aiohttp` 初始化 SSL context 时在 Windows 10 上加载证书存储失败。当前环境已有 Ollama bge-m3，应使用 `api` 模式。

**修复**：`.env` 中设置 `EMBEDDING_PROVIDER=api`，使用 Ollama 的 OpenAI 兼容 API。

**注意**：如果将来需要本地 embedding（无网络/Ollama），需解决 `sentence-transformers` 的 SSL 问题，或使用 `pip install sentence-transformers --no-deps` 后手动安装不依赖 `aiohttp` 的版本。

---

## 问题 4：前端 JS 格式不兼容分页后的 API 响应

**现象**：API 返回 `{"data":{"items":[...],"total":N,"page":1}}`，但前端 JS 仍用旧格式 `data.knowledge_bases`、`data.documents`、`data.records` 读取，导致页面上知识库列表、文档列表、考试记录全部为空。

**原因**：P0-6 分页改造后，API 响应格式从 `{"knowledge_bases": [...]}` 变为 `{"items": [...], "total": N, ...}`，但前端 5 个 HTML 模板的 JS 代码未同步更新。

**修复**：在以下 5 个模板中，将旧字段改为兼容写法 `data.items || data.knowledge_bases`：

| 模板 | 修改点 |
|------|--------|
| `app/templates/knowledge.html` | `data.knowledge_bases` → `data.items \|\| data.knowledge_bases` |
| `app/templates/qa.html` | 同上（知识库下拉选择器） |
| `app/templates/upload.html` | 知识库选择器 + 文档列表 `data.documents` → `data.items \|\| data.documents` |
| `app/templates/exam_generate.html` | 知识库选择器 + 考试记录列表 `data.records` → `data.items \|\| data.records` |

**Python 3.10 兼容性补充**：`datetime.timezone` 仅在 Python 3.11+ 中可用，`documents.py` 中误用导致 `AttributeError`，已回退为 `datetime.utcnow()`。
