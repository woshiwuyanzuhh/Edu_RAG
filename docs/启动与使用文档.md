## 一、架构与端口总览

### 服务拓扑

```
浏览器 → 前端(5173) ──→ 后端 app(8000, 4 workers)
                          ├── ARQ worker（文档异步入库）
                          ├── MySQL(3307)
                          ├── Redis(6380)
                          ├── Milvus(19530) ← 宿主机已运行
                          └── Ollama(11434) ← 宿主机已运行
```

### 端口与凭证

| 服务 | 端口 | 凭证 | 说明 |
|---|---|---|---|
| 后端 API | 8000 | API Key: `dev_test_key_2026` | docker-app 容器 |
| ARQ worker | - | - | docker-worker 容器（异步入库） |
| MySQL | 3307 | 密码见 `.env` 的 `MYSQL__PASSWORD` | docker-mysql 容器 |
| Redis | 6380 | 密码: `redis_dev_2026` | docker-redis 容器 |
| Milvus | 19530 / 9091 | 无密码 | 宿主机 milvus-standalone 容器 |
| Ollama | 11434 | 无 | 宿主机 Ollama（bge-m3 模型） |
| 前端 | 5173 | - | vite 开发服务器 |

> **注意**：宿主机还有一套 Milvus 生态（milvus-standalone + milvus-redis + milvus-etcd + milvus-minio），端口 6379/19530/9091。docker 的 redis 用 6380 避开冲突。

## 二、方式一：Docker 启动（推荐）

### 前置条件

- Docker Desktop 已安装并运行
- 宿主机已运行：Milvus（`milvus-standalone` 容器）+ Ollama（bge-m3 模型）
- `.env` 文件已配置（含 `MYSQL_PASSWORD` / `REDIS_PASSWORD` / `OPENAI_API_KEY` / `APP_API_KEY`）

### 2.1 启动后端（docker compose）

```bash
cd C:/Users/lenovo/Desktop/ml_dl_nlp/edu_rag

# 启动（app + worker + mysql + redis，复用宿主机 Milvus + Ollama）
docker compose -f docker/docker-compose.yml -f docker/docker-compose.override.yml --env-file .env up -d
```

**启动后会创建 4 个容器**：
- `docker-app-1`：后端 API（8000）
- `docker-worker-1`：ARQ 异步任务 worker
- `docker-mysql-1`：MySQL（3307）
- `docker-redis-1`：Redis（6380）

**验证启动成功**：
```bash
# 查看容器状态（应全部 Up）
docker compose -f docker/docker-compose.yml -f docker/docker-compose.override.yml ps

# 健康检查（应返回 status:ok，三项全 true）
curl http://localhost:8000/health
# 预期：{"status":"ok","checks":{"mysql":true,"redis":true,"vector_store":true}}
```

**查看日志**：
```bash
# 实时查看所有日志
docker compose -f docker/docker-compose.yml -f docker/docker-compose.override.yml logs -f

# 只看 app 日志
docker logs -f docker-app-1

# 只看 worker 日志（文档入库处理）
docker logs -f docker-worker-1
```

### 2.2 启动前端（vite 开发服务器）

```bash
cd C:/Users/lenovo/Desktop/ml_dl_nlp/edu_rag/frontend

# 首次需安装依赖（node_modules 已存在则跳过）
npm install

# 启动开发服务器
npm run dev
```

启动后访问：**http://localhost:5173**

> 前端 vite 服务器会自动代理 API 请求到 8000 端口（见 `frontend/vite.config.ts` 的 proxy 配置）。

### 2.3 停止服务

```bash
cd C:/Users/lenovo/Desktop/ml_dl_nlp/edu_rag

# 停止容器（保留数据）
docker compose -f docker/docker-compose.yml -f docker/docker-compose.override.yml down

# 停止并删除数据卷（清空 MySQL/Redis 数据，慎用）
docker compose -f docker/docker-compose.yml -f docker/docker-compose.override.yml down -v

# 前端：在 npm run dev 的终端按 Ctrl+C
```

### 2.4 重新构建镜像（改了 Dockerfile 或依赖后）

```bash
docker compose -f docker/docker-compose.yml -f docker/docker-compose.override.yml --env-file .env up -d --build
```

> **代码热更新**：`docker-compose.override.yml` 已挂载 `../src:/app/src`，改 Python 代码后重启容器即可生效（不用重建镜像）：`docker compose -f docker/docker-compose.yml -f docker/docker-compose.override.yml restart app worker`

## 三、方式二：PyCharm 启动（本地开发）

### 前置条件

- PyCharm 已安装
- Conda 环境 `EduRAG` 已创建（路径：`F:\anaconda3\envs\EduRAG`）
- 宿主机已运行：MySQL（3306 或 docker 3307）、Redis（6379 或 docker 6380）、Milvus（19530）、Ollama（11434）

### 3.1 配置 .env

确保项目根目录 `.env` 配置正确（关键变量）：

```ini
# LLM
LLM__API_KEY=sk-你的DeepSeek密钥
LLM__BASE_URL=https://api.deepseek.com/v1
LLM__MODEL=deepseek-chat

# Embedding（用宿主机 Ollama）
EMBEDDING__PROVIDER=api
EMBEDDING__API_BASE_URL=http://localhost:11434/v1
EMBEDDING__MODEL=bge-m3
EMBEDDING__API_KEY=ollama

# 向量库（用宿主机 Milvus）
VECTOR_STORE__PROVIDER=milvus
VECTOR_STORE__MILVUS_HOST=localhost
VECTOR_STORE__MILVUS_PORT=19530

# MySQL（用宿主机或 docker 的）
MYSQL__HOST=localhost
MYSQL__PORT=3306
MYSQL__USER=root
MYSQL__PASSWORD=你的MySQL密码
MYSQL__DATABASE=edu_rag

# Redis
REDIS__HOST=localhost
REDIS__PORT=6379
REDIS__PASSWORD=你的Redis密码

# 应用
APP__HOST=0.0.0.0
APP__PORT=8000
APP__API_KEY=dev_test_key_2026
APP__ASYNC_INGESTION=true
```

> **注意**：PyCharm 本地启动时，用 `localhost` 连本地服务（不是 docker 的 `host.docker.internal`）。如果你用 docker 的 MySQL/Redis，端口改为 3307/6380。

### 3.2 安装 Python 依赖

```bash
# 激活 Conda 环境
conda activate EduRAG

# 安装依赖（用锁文件确保版本一致）
pip install -r requirements.txt
```

### 3.3 PyCharm 运行配置

#### 配置 1：后端 API

1. PyCharm → Run → Edit Configurations → `+` → Python
2. 填写：
   - **Name**: `edu_rag API`
   - **Script path**: 选择项目根目录的 `src/orchestration/app.py`（或用 Module name: `uvicorn`）
   - **Parameters**: `src.orchestration.app:app --host 0.0.0.0 --port 8000 --workers 4`（如果用 module 方式）
   - **Python interpreter**: 选择 Conda `EduRAG`
   - **Working directory**: `C:/Users/lenovo/Desktop/ml_dl_nlp/edu_rag`

   或者更简单——用 **Module name** 方式：
   - **Module name**: `uvicorn`
   - **Parameters**: `src.orchestration.app:app --host 0.0.0.0 --port 8000 --reload`
   - **Working directory**: 项目根目录

3. 点 OK 保存

#### 配置 2：ARQ Worker（异步任务，可选）

如果 `APP__ASYNC_INGESTION=true`，需要单独起 worker：

1. `+` → Python
2. **Module name**: `arq`
3. **Parameters**: `src.orchestration.worker.config.WorkerSettings`
4. **Working directory**: 项目根目录
5. **Python interpreter**: Conda `EduRAG`

> 如果不想起 worker，设 `APP__ASYNC_INGESTION=false`，文档入库会同步处理（开发简单模式）。

### 3.4 启动前端

```bash
cd C:/Users/lenovo/Desktop/ml_dl_nlp/edu_rag/frontend
npm run dev
```

### 3.5 启动顺序

1. 确保宿主机 MySQL / Redis / Milvus / Ollama 已运行
2. PyCharm 运行 `edu_rag API` 配置
3. （可选）PyCharm 运行 ARQ Worker 配置
4. 终端运行 `npm run dev`
5. 浏览器访问 http://localhost:5173

## 四、API 鉴权

### 4.1 鉴权机制

系统使用 **API Key 鉴权**（fail-closed：未配置密钥时拒绝所有请求）。

- API Key 配置在 `.env` 的 `APP__API_KEY`（docker 用 `APP_API_KEY`）
- 当前 Key: `dev_test_key_2026`

### 4.2 请求头携带 Key

所有 `/api/*` 请求需在 Header 携带：

```
X-API-Key: dev_test_key_2026
```

**curl 示例**：
```bash
# 创建知识库
curl -X POST http://localhost:8000/api/kb \
  -H "Content-Type: application/json" \
  -H "X-API-Key: dev_test_key_2026" \
  -d '{"name":"测试库","description":"测试"}'

# 查询知识库列表
curl -H "X-API-Key: dev_test_key_2026" http://localhost:8000/api/kb

# 上传文档
curl -X POST http://localhost:8000/api/documents/upload \
  -H "X-API-Key: dev_test_key_2026" \
  -F "file=@data/1_test_ai_knowledge.txt" \
  -F "knowledge_base_id=1" \
  -F "doc_type=general"

# QA 问答
curl -X POST http://localhost:8000/api/qa \
  -H "Content-Type: application/json" \
  -H "X-API-Key: dev_test_key_2026" \
  -d '{"question":"人工智能是什么","knowledge_base_id":1}'
```

### 4.3 Swagger UI 鉴权

1. 浏览器打开 **http://localhost:8000/docs**
2. 点击页面右上角 **Authorize** 按钮（🔒 图标）
3. 在输入框填入 `dev_test_key_2026`
4. 点 Authorize 确认
5. 之后在 Swagger 页面直接点 "Try it out" 测试任何 API（自动带 Key）

### 4.4 前端鉴权

前端（5173）启动后，在页面上的"设置"或"API Key"输入框填入 `dev_test_key_2026`，前端会自动在请求头携带。

#### 方式 B：Python 查询

```bash

from pymilvus import Collection, connections
connections.connect(host='host.docker.internal', port=19530)
c = Collection('edu_docs')
c.load()
print('总向量数:', c.num_entities)
# 查询数据
res = c.query(expr='knowledge_base_id == 1', output_fields=['text','doc_id'], limit=5)
for r in res:
    print('doc_id:', r.get('doc_id'), 'text:', str(r.get('text',''))[:60])
"
```

- **字段**: id（VARCHAR 主键）、text、embedding（1024 维 FLOAT_VECTOR）、knowledge_base_id、doc_id、chunk_index
- **索引**: HNSW（M=16, efConstruction=200），COSINE 相似度

## 七、常见问题

### Q1：首页 http://localhost:8000/ 返回"未授权"

**原因**：AuthMiddleware 拦截所有请求（含前端静态文件）。

**解决**：用前端 dev server（`npm run dev` → 5173），或修改 AuthMiddleware 排除 `GET /` 和 `/assets/*`。

### Q2：上传文档后 QA 检索为空

**可能原因**：
1. worker 还在处理（查 `docker logs docker-worker-1`）
2. Milvus collection 需 reload（app 启动后 worker 写入的数据）：重启 app 容器 `docker compose restart app`
3. min_score 阈值过高（默认 0.3，可在 `.env` 调 `RETRIEVAL__MIN_SCORE=0.1`）

### Q3：docker compose up 报 "REDIS_PASSWORD is missing"

**原因**：`.env` 没有单下划线变量（docker compose 用单下划线，pydantic 用双下划线）。

**解决**：确保 `.env` 同时有：
```ini
MYSQL__PASSWORD=xxx    # pydantic 用
MYSQL_PASSWORD=xxx     # docker compose 用
REDIS__PASSWORD=xxx
REDIS_PASSWORD=xxx
APP__API_KEY=xxx
APP_API_KEY=xxx
```

### Q4：PyCharm 启动报 "ModuleNotFoundError: arq"

**解决**：`pip install arq pymilvus`（或 `pip install -r requirements.txt`）

### Q5：Milvus 连接失败

**检查**：
```bash
curl http://localhost:9091/healthz   # 应返回 OK
docker ps | grep milvus              # milvus-standalone 应 Up
```

### Q6：worker 不断重启

**查日志**：`docker logs docker-worker-1`
常见原因：MySQL 认证方式（需 native_password）、Milvus 连接失败、依赖缺失。
